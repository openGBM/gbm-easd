# Technical Role — Technical Environment Completion Gate

Runs once every question under `## Technical Questions` in `aidlc-discovery-state.md` is marked `[x]` (or the user has explicitly waived remaining questions).

---

## Step 1 — Run the open-questions collector

Execute `shared/open-questions-collector.md` for the Technical role. This updates `Product-Definition/open-questions.md` (merging with the Business open questions already there, if any).

## Step 2 — Render the Technical Environment Document

Read:
- `Product-Definition/interview/role-selection.md`
- `Product-Definition/interview/project-type.md`
- `Product-Definition/vision-document.md` (if present — used only for cross-references in prose)
- `Product-Definition/interview/technical/tech-env-answers-history.md`
- `Product-Definition/open-questions.md`

Write `Product-Definition/technical-environment.md` using the structure below. All sections are mandatory; if one has no content, include a short "Not applicable — <reason>" line.

## Step 3 — Validate the rendered document

Apply `common/content-validation.md`. Specifically for Technical:

- All "Required" tables have at least one row. If they are empty, ask the user to confirm the project truly has no required frameworks — very rare.
- Prohibited libraries table: every row has a reason AND an alternative.
- Example code blocks compile mentally — if a snippet is obviously broken (e.g. unterminated string, mismatched braces), flag for fix before rendering.
- Security sections align with the chosen compliance framework. If `T21 = HIPAA` but encryption is not "at rest and in transit", flag as a contradiction.

If any check fails, do not present the gate — ask the user to resolve before rendering.

## Step 4 — Present the completion gate

Exactly two options:

```
Technical Environment Document complete — review before we hand off.

  Product-Definition/technical-environment.md is ready.
  Product-Definition/open-questions.md has been refreshed.

Your options:

  1) Request Changes — tell me what to fix; I'll update the answer history
     and re-render.

  2) Approve and Continue — mark Technical as ✅ complete and move to
     the AI-DLC Discovery handoff.

What would you like to do?
```

Wait for the user's explicit choice.

## Step 5 — Handle the user's choice

- **Request Changes** → pinpoint the section, append a correction to `tech-env-answers-history.md` (never overwrite), re-render, re-validate, re-present.
- **Approve and Continue** →
  1. Update `aidlc-discovery-state.md`:
     ```markdown
     ## Role Progress
     - Technical (Technical Env Doc):  ✅ Complete (approved <ISO8601>)
     ```
  2. Log approval in `audit.md` with stage label `Technical Interview — Completion`.
  3. Transition to the Final Handoff described in `aidlc-discovery-core-workflow.md`. The handoff message presents the three required AI-DLC inputs (`vision-document.md`, `technical-environment.md`, `open-questions.md`) plus, when Visual Sketch was approved, the optional `visual/` sidecar. Use the exact template in `aidlc-discovery-core-workflow.md` §"Final Handoff".

## Step 6 — Log everything

Every interaction in this gate is logged per `common/audit-format.md`.
