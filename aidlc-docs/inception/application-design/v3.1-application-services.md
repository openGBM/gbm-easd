# Application Services (Use Cases) — Abstracción Arquitectónica v3.1

## ADR-006: Capa de Application Services

| Aspecto | Detalle |
|---------|---------|
| **Contexto** | Los componentes tienen operaciones multi-paso complejas (duplicar instrumentos, crear sesiones con límites, cargar dashboards con stats, etc.) que no encajan en repos thin |
| **Decisión** | Agregar capa `core/services/` con Use Cases que orquestan múltiples repos |
| **Alternativas** | Fat repositories (lógica en repos), lógica en componentes (status quo) |
| **Consecuencias** | (+) Repos thin y testables, (+) lógica de negocio centralizada, (+) error handling consistente, (-) más archivos |

---

## Estructura

```
src/core/
├── services/                        # 🆕 Application Services (Use Cases)
│   ├── session.service.ts           # Crear sesión con límites, stats, CRUD completo
│   ├── instrument.service.ts        # Duplicar instrumento, crear con versión
│   ├── survey.service.ts            # Flujo encuesta: cargar dimensiones, guardar respuestas
│   ├── analytics.service.ts         # Dashboard stats, tendencias, historial encuestados
│   ├── respondent.service.ts        # Registrar, reanudar, eliminar con cascade
│   └── index.ts                     # Barrel export
```

## Services Identificados

### 1. SessionService

Orquesta operaciones de sesiones que involucran múltiples repos + validaciones.

```typescript
interface SessionService {
  // Crear sesión verificando límite del tenant + seleccionando versión del instrumento
  createSession(data: CreateSessionInput): Promise<Result<Session, DomainError>>
  
  // Cargar sesiones con conteo de respondents + info de instrumento
  listSessionsWithStats(filters?: SessionFilters): Promise<Result<SessionWithStats[], DomainError>>
  
  // Dashboard stats: sesiones activas, total respondents, avg time, total instruments
  getDashboardStats(): Promise<Result<DashboardStats, DomainError>>
  
  // Eliminar sesión con cascade (respondents + responses)
  deleteSessionCascade(sessionId: string): Promise<Result<void, DomainError>>
  
  // Cargar detalle de sesión con respondents + instrument info + maturity levels
  getSessionDetail(sessionId: string): Promise<Result<SessionDetail, DomainError>>
}
```

### 2. InstrumentService

Orquesta operaciones complejas de instrumentos.

```typescript
interface InstrumentService {
  // Crear instrumento + versión inicial
  createInstrument(data: CreateInstrumentInput): Promise<Result<InstrumentWithVersion, DomainError>>
  
  // Duplicar instrumento completo (instrumento + versión + dimensiones + preguntas)
  duplicateInstrument(sourceId: string, newName: string): Promise<Result<InstrumentWithVersion, DomainError>>
  
  // Listar instrumentos con conteo de sesiones por versión
  listInstrumentsWithStats(): Promise<Result<InstrumentWithStats[], DomainError>>
  
  // Toggle activo/inactivo
  toggleActive(id: string): Promise<Result<void, DomainError>>
}
```

### 3. SurveyService

Orquesta el flujo de encuesta (server-side page + client form).

```typescript
interface SurveyService {
  // Cargar todo lo necesario para la página de encuesta
  loadSurveyPage(sessionId: string): Promise<Result<SurveyPageData, DomainError>>
  
  // Guardar respuestas (numéricas + texto, con fallback)
  saveResponses(respondentId: string, responses: SaveResponseInput[]): Promise<Result<void, DomainError>>
  
  // Marcar encuesta como completada
  completeSubmission(respondentId: string): Promise<Result<void, DomainError>>
  
  // Cargar respuestas previas para reanudación
  loadPreviousResponses(respondentId: string): Promise<Result<PreviousResponses, DomainError>>
}
```

### 4. AnalyticsService

Orquesta queries de analytics y consolidación.

```typescript
interface AnalyticsService {
  // Tendencias de todas las sesiones de un instrumento
  getTrends(instrumentId: string, filters?: TrendFilters): Promise<Result<TrendData, DomainError>>
  
  // Historial consolidado de un encuestado por email
  getRespondentHistory(email: string): Promise<Result<RespondentHistory, DomainError>>
  
  // Scores consolidados de una sesión (para radar chart)
  getConsolidatedScores(sessionId: string): Promise<Result<DimensionScore[], DomainError>>
  
  // Scores individuales de un respondent
  getIndividualScores(respondentId: string): Promise<Result<DimensionScore[], DomainError>>
}
```

### 5. RespondentService

Orquesta registro y operaciones sobre encuestados.

```typescript
interface RespondentService {
  // Registrar o reanudar encuestado (lógica de upsert + verificación sesión activa)
  registerOrResume(data: RegisterRespondentInput): Promise<Result<RegisterResult, DomainError>>
  
  // Eliminar encuestado con cascade (responses)
  deleteCascade(respondentId: string): Promise<Result<void, DomainError>>
}
```

## Tipos de Entrada/Salida

```typescript
// Session
interface CreateSessionInput {
  name: string
  instrumentId?: string  // resuelve la versión current automáticamente
  tenantId?: string
}

interface SessionWithStats extends Session {
  respondentCount: number
  instrumentName?: string
  versionTag?: string
}

interface DashboardStats {
  activeSessions: number
  totalCompleted: number
  avgTimeMinutes: number
  totalInstruments: number
}

interface SessionDetail extends Session {
  respondents: Respondent[]
  instrumentInfo?: { name: string; versionTag: string }
  maturityLevels?: unknown[]
  stats: { totalCompleted: number; avgTimeMinutes: number }
}

// Instrument
interface CreateInstrumentInput {
  name: string
  description?: string
  aiExpertisePrompt?: string
}

interface InstrumentWithStats extends InstrumentWithVersion {
  sessionCount: number
}

// Survey
interface SurveyPageData {
  session: Session
  dimensions: DimensionWithQuestions[]
  scaleLabels?: unknown[]
  instrumentName: string
  instrumentDescription?: string
  totalQuestions: number
  estimatedMinutes: number
}

interface SaveResponseInput {
  questionId: string
  value: number | null
  textValue?: string | null
}

interface PreviousResponses {
  numeric: Record<string, number>
  text: Record<string, string>
}

// Analytics
interface TrendData {
  dataPoints: TrendDataPoint[]
  dimensions: DimensionInfo[]
  sessions: { id: string; name: string; date: string }[]
}

interface RespondentHistory {
  email: string
  sessions: RespondentSession[]
}

// Respondent
interface RegisterRespondentInput {
  sessionId: string
  name: string
  email: string
}

interface RegisterResult {
  respondentId: string
  resumed: boolean
}
```
