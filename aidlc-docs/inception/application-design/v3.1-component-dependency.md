## 6. Reglas de Dependencia (Dependency Rules)

| Regla | Descripción | Validación |
|-------|-------------|------------|
| **DR-01** | Consumers NUNCA importan Adapters ni vendor SDKs | grep para `@supabase`, `@google`, `groq-sdk` fuera de `core/adapters/` |
| **DR-02** | Ports NUNCA importan Adapters | grep para imports relativos hacia `../adapters/` en `core/ports/` |
| **DR-03** | Adapters NUNCA importan otros Adapters | grep para imports cruzados entre `adapters/supabase/` y `adapters/ai/` |
| **DR-04** | Solo Container importa Adapters | Todos los imports de `core/adapters/` deben estar en `core/server-container.ts` o `core/client-container.ts` |
| **DR-05** | Cross-cutting es framework-agnostic | Logger, Metrics, DomainError no importan Next.js ni vendor SDKs |
| **DR-06** | Ports son estables | Cambiar un port requiere actualizar TODOS los adapters (contract tests lo detectan) |
| **DR-07** | Services solo importan Ports, nunca Adapters | `core/services/` importa de `core/ports/` y `core/errors/`, nunca de `core/adapters/` |
| **DR-08** | Services se instancian via helper factories (`createXxxService(container)`), no directamente ni en Container | Las factories están en `core/services/factories.ts` |
