# Métodos de Componentes — Abstracción Arquitectónica v3.1

## Convenciones

- Todos los métodos retornan `Promise<Result<T, DomainError>>` (discriminated union, no excepciones)
- Los tipos de entrada usan Zod schemas para validación
- Los métodos de escritura reciben DTOs, no entidades completas
- Nota: Las reglas de negocio detalladas se definen en Functional Design (per-unit)

## 6. InstrumentedRepository (Decorator) — Métodos

```typescript
// Decorator genérico que envuelve cualquier repository con observabilidad
class InstrumentedRepository<T> implements T {
  constructor(
    inner: T,
    logger: Logger,
    metrics: MetricsCollector,
    repositoryName: string
  )
  // Intercepta cada método del inner repository y:
  // 1. Registra inicio (debug log)
  // 2. Mide duración
  // 3. Registra resultado (info/error log)
  // 4. Emite métricas (latencia, errores)
}
```

## 7. DomainError — Tipos

```typescript
// Base
abstract class DomainError {
  abstract readonly code: string
  abstract readonly message: string
  abstract readonly httpStatus: number
}

// Específicos
class NotFoundError extends DomainError { code: 'NOT_FOUND'; httpStatus: 404 }
class ConflictError extends DomainError { code: 'CONFLICT'; httpStatus: 409 }
class UnauthorizedError extends DomainError { code: 'UNAUTHORIZED'; httpStatus: 401 }
class ForbiddenError extends DomainError { code: 'FORBIDDEN'; httpStatus: 403 }
class ValidationError extends DomainError { code: 'VALIDATION'; httpStatus: 400 }
class RateLimitError extends DomainError { code: 'RATE_LIMIT'; httpStatus: 429 }
class ServiceUnavailableError extends DomainError { code: 'SERVICE_UNAVAILABLE'; httpStatus: 503 }
class InternalError extends DomainError { code: 'INTERNAL'; httpStatus: 500 }

// Result type (discriminated union)
type Result<T, E extends DomainError = DomainError> =
  | { ok: true; value: T }
  | { ok: false; error: E }
```
